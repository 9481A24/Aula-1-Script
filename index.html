🔷 Script Termux – Scanner RTSP Câmeras IP

📖 1. Instale o Nmap e Curl

No Termux, digite:

pkg update
pkg install nmap
pkg install curl

📝 2. Crie o script

No Termux, digite:

nano rtsp_scan.sh

Cole esse código completo:

#!/data/data/com.termux/files/usr/bin/bash

echo "🔷 Scanner RTSP – Buscando câmeras IP na rede..."

# Descobrir o gateway padrão
gateway=$(ip route | grep default | awk '{print $3}')
echo "Gateway: $gateway"

# Descobrir a faixa de rede
rede=$(echo $gateway | cut -d'.' -f1-3).0/24
echo "Rede: $rede"

# Escanear IPs com porta 554 aberta (RTSP)
echo "🔷 Escaneando IPs ativos na rede $rede"
ips=$(nmap -p 554 --open $rede | grep "Nmap scan report for" | awk '{print $5}')

# Testar RTSP em cada IP encontrado
for ip in $ips
do
  echo ""
  echo "📹 Testando RTSP no IP: $ip"
  timeout 5 curl -v rtsp://$ip:554 --stderr - | grep -i "rtsp"
done

echo ""
echo "✅ Scanner RTSP concluído."

✅ 3. Salve e execute

  • Para salvar no nano:
    ➔ Pressione CTRL + O e Enter
  • Para sair do nano:
    ➔ Pressione CTRL + X
  • Dê permissão de execução:
    chmod +x rtsp_scan.sh
  • Execute o script:
    ./rtsp_scan.sh
⚠️ Dicas importantes:
  • Use apenas em redes onde você tem autorização.
  • Câmeras IP geralmente usam porta 554 (RTSP).
  • Necessário usuário e senha para acessar o vídeo.
  • Para treinamento, use URLs RTSP de teste em laboratórios virtuais.

✅ Passo a passo rápido

1. Atualize e instale o Nmap
No Termux:

pkg update && pkg upgrade
pkg install nmap

✅ Copie e cole este script direto no Termux

nmap -sn 192.168.1.0/24
echo "==== Scan de portas RTSP de câmeras IP ===="
nmap -p 554 --open 192.168.1.0/24
echo "==== Scan rápido de portas comuns ===="
nmap -p 22,80,443,445 --open 192.168.1.0/24

🔍 O que esse script faz?

✅ Linha 1: Lista todos dispositivos conectados no seu Wi-Fi (ping scan).
✅ Linha 2-3: Busca dispositivos com porta 554 aberta (RTSP), geralmente câmeras IP.
✅ Linha 4-5: Verifica portas mais comuns abertas (SSH, HTTP, HTTPS, SMB).

✅ 1. Primeiro: O que significa porta aberta ou fechada?

🔓 Porta aberta:
- O dispositivo está ouvindo conexões naquela porta.
- Exemplo: Porta 554 aberta → provavelmente tem câmera IP com RTSP ativo.

🔒 Porta fechada:
- Não está aceitando conexões externas nessa porta naquele momento.

🔍 2. O que fazer quando encontra porta aberta?

✔ A. Identificar o serviço
Use:
nmap -sV -p [porta] [ip]

Exemplo:
nmap -sV -p 554 192.168.1.10

Isso mostra qual serviço está rodando, versão, etc.

✔ B. Tentar conectar ao serviço

- Porta 554 (RTSP) → usar apps como VLC:
  rtsp://192.168.1.10:554

- Porta 80 ou 443 (HTTP/HTTPS) → abrir no navegador.

- Porta 22 (SSH) → conectar com:
  ssh usuario@192.168.1.10

✔ C. Analisar a segurança

Perguntas:
1. Esse serviço deveria estar aberto?
2. Tem autenticação forte?
3. Está atualizado ou tem falhas conhecidas?

⚠️ 3. Muito importante – Ética e Legislação

✔ Somente teste redes que você tenha autorização (sua, laboratório, estágio).
❌ Tentar acesso sem autorização é crime de invasão (Lei 12.737/2012 – Brasil).
